Lab 3: Hardware vs Software Transactional Memory

Variant 15 | DBMS: PostgreSQL | API: java.sql (JDBC) | Threads: 8

Experiment: 8 threads concurrently execute UPDATE queries on a PostgreSQL database. The number of queries per thread varies to produce different total query counts. We compare two approaches to managing concurrent access:
  • HTM (Hardware Transactional Memory): ReentrantLock with Intel TSX-NI Hardware Lock Elision
  • STM (Software Transactional Memory): Multiverse library atomic blocks
HTM (Hardware Transactional Memory): Uses java.util.concurrent.locks.ReentrantLock to protect concurrent DB operations. When Intel TSX-NI is enabled, the CPU applies Hardware Lock Elision (HLE): lock-protected regions are speculatively executed without actually acquiring the lock. Since threads operate on largely non-overlapping records, speculative execution succeeds and lock overhead is eliminated.
STM (Software Transactional Memory): Uses the Multiverse library (StmUtils.atomic()) to wrap DB operations in software-managed transactional blocks. Conflict detection and resolution are managed entirely in software, adding overhead compared to hardware-assisted approach. This mode is run with TSX-NI disabled to measure pure STM performance.

Conclusions

When Intel TSX-NI is enabled, the CPU uses Hardware Lock Elision (HLE) to speculatively execute critical sections protected by ReentrantLock without actually acquiring the lock. Hardware transactional memory manages conflicts at the CPU level with minimal overhead.

Software Transactional Memory (Multiverse) provides the same transactional guarantees but with higher overhead due to software-based versioning, conflict detection, and potential rollback/retry. Each StmUtils.atomic() call adds bookkeeping overhead.

The STM/HTM ratio shows how much slower STM is compared to HTM. A ratio > 1.0 means STM is slower. The overhead tends to grow with the total number of queries as more STM bookkeeping accumulates.

TSX-NI Control (Windows Registry)

Enable TSX-NI:
reg add "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Kernel" /v DisableTsx /t REG_DWORD /d 0 /f

Disable TSX-NI:
reg add "HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\Session Manager\Kernel" /v DisableTsx /t REG_DWORD /d 1 /f

Reboot required after each change.
